## 4. Duplication candidates (Solo ↔ MP ↔ Host ↔ Player)

1. **Answer-screen orchestration — `states/Answers.tsx` (724L, MP) vs `states/SoloAnswers.tsx` (431L, Solo).** Self-documented dup (`SoloAnswers.tsx:1-9`: "Mirrors the layout of Answers.tsx but uses REST instead of socket.emit"). Leaf components (`ChoiceGrid`, `MultiSelectGrid`, `MathematikInput`, `SentenceBuilderBoard`, `SliderInput`, `TypeAnswerInput`, `WortartenPicker`, `CircularTimer`, `QuestionMedia`, `Markdown`) are **already correctly shared** — the duplication is confined to the transport/timer/feedback orchestration shell. Highest-value target for #21 modularization (e.g. extract a transport-agnostic answer-orchestration hook that both wrap).
2. **Join / name-entry flow — `join/Username.tsx` (MP, uses shared `Input`/`Button`/`Card`) vs `solo/SoloNameScreen.tsx` (Solo, raw `<input>`/`<button>` with independent gradient styling, `SoloNameScreen.tsx:45-60`).** Not just duplicated logic — the Solo version **doesn't consume the shared primitives at all**, so any future primitive-level design change (e.g. focus-ring fix, token swap) silently misses Solo. Also directly relevant to charter item 4 (Klassenmodus name-select): whichever component becomes `PlayerNameSelect` needs a single home, not two divergent name-entry implementations.
3. **Dialog primitive fragmentation.** `components/AlertDialog.tsx` (shared, used by 14 manager/quizz files) vs. three **game-surface** files that hand-roll raw `@radix-ui/react-alert-dialog` instead: `pages/party/manager/$gameId.tsx`, `features/game/components/GameWrapper/RejoinQrDialog.tsx:1`, `features/game/components/states/Room.tsx:1`. Not a Solo/MP split — a game-vs-manager-console split, but squarely in this inventory's target scope (`AlertDialog` was an explicit target primitive).
4. **Score/points display — 5+ independent implementations**, no `ScoreBadge` primitive: `SoloShell.tsx:100-108`, `states/Paused.tsx`, `states/Responses.tsx`, `states/Result.tsx` (via `RewardStack`), `SoloLeaderboard.tsx`. Each restyles the same "number in a rounded pill" concept slightly differently (see §1 ScoreBadge row).
